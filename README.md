# Algebraic Reasoning Distiller

A multi-agent pipeline that decides whether one equational law implies another over all **magmas** (sets equipped with a single binary operation `*`, with no axioms assumed). Built for the **[SAIR Foundation — Mathematics Distillation Challenge, Stage 1: Equational Theories](https://sair.foundation)** competition.

The pipeline combines symbolic reasoning with a fine-tuned LLM distiller to produce a compact ≤10 KB *cheat sheet* injected into the offline judge's prompt.

---

## Problem

Given two equations over a binary operation `*`, determine whether:

> **E1 ⟹ E2** holds in every magma

**TRUE** — every magma satisfying E1 also satisfies E2.  
**FALSE** — there exists a finite magma where E1 holds but E2 does not (counterexample).

The challenge: the judge evaluates submissions offline with no tool access, ≤10 minutes and ≤$0.01 per problem. The only allowed resource is a ≤10 KB static cheat sheet prepended to the judge's context.

## Architecture

```
                         ┌──────────────────────────────────┐
         E1, E2 ──────►  │         Planner (parser)          │
                         └────────────┬─────────────────────┘
                                      │ parsed AST
              ┌───────────────────────┼────────────────────┐
              ▼                       ▼                     ▼
    ┌──────────────────┐   ┌──────────────────┐   ┌──────────────────┐
    │  Counterexample  │   │  Symbolic Prover │   │  RAG Retriever   │
    │  (magma search)  │   │  (term rewriting)│   │  (Chroma + SBERT)│
    └────────┬─────────┘   └────────┬─────────┘   └────────┬─────────┘
             │                      │                       │
             └──────────────────────┼───────────────────────┘
                                    ▼
                         ┌──────────────────────┐
                         │      Aggregator       │
                         │  (counterex > proof   │
                         │   > retrieved)        │
                         └──────────┬───────────┘
                                    │ EvidenceBundle
                                    ▼
                         ┌──────────────────────┐
                         │  LLM Distiller        │
                         │  Qwen2.5-7B + LoRA    │
                         │  (SFT → GRPO)         │
                         └──────────┬───────────┘
                                    │
                                    ▼
                         ┌──────────────────────┐
                         │  Cheat Sheet (≤10KB)  │
                         │  sair/cheat_sheets/   │
                         └──────────────────────┘
```

### Agents

| Agent | Description |
|---|---|
| **Counterexample** | Exhaustive search over magmas of order ≤3; random sampling for order 4. Hard-FALSE verdict when found. |
| **Symbolic Prover** | Alpha-equivalence, substitution instance, one-step term rewriting. Hard-TRUE verdict (confidence 0.9). |
| **RAG Retriever** | Chroma vector store of canonical equations + known implications. Uses `all-MiniLM-L6-v2` embeddings. |
| **Aggregator** | Fuses evidence with priority: counterexample > symbolic proof > retrieved context. |
| **LLM Distiller** | `Qwen/Qwen2.5-7B-Instruct` fine-tuned with LoRA via SFT then GRPO. Reward = official `judge_response` verdict correctness + format bonuses. |

### Training

1. **SFT** — supervised fine-tuning on evidence-backed `<think>/<answer>` completions generated by the offline symbolic agents.
2. **GRPO** — group relative policy optimization where the reward function calls the *official* SAIR `judge_response` extractor, ensuring the training signal is byte-identical to the grader.

## Quick start (DGX cluster)

### 1. Boot the container

```bash
cd container/
nano variables.sh          # set CONTAINER_USER to your DGX username

chmod +x up.sh attach.sh down.sh
./up.sh                    # build + start (first time ~5 min)
./attach.sh                # open shell inside the container
```

### 2. Clone this repo (inside container, under ~/data/)

```bash
cd ~/data
register-github-repo --remote=git@github.com:<your-user>/Algebraic-Reasoning-Distiller.git
# follow the instructions to add the deploy key to GitHub, then clone

# Clone the SAIR judge repo as a sibling (required for reward alignment)
git clone https://github.com/sair-foundation/equational-theories-stage1-judge.git
```

Expected layout on disk:

```
~/data/
  Algebraic-Reasoning-Distiller/   ← this repo
  equational-theories-stage1-judge/ ← sibling (needed by evaluator.py)
```

### 3. Install dependencies

```bash
cd ~/data/Algebraic-Reasoning-Distiller
uv sync

# Install PyTorch with CUDA (adjust cu121 → your DGX CUDA version)
uv pip uninstall torch torchvision torchaudio
uv pip install torch torchvision torchaudio \
    --index-url https://download.pytorch.org/whl/cu121

# Verify GPU access
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.device_count())"
```

### 4. Download base model

```bash
huggingface-cli login        # paste your HF token (read-only is enough)
huggingface-cli download Qwen/Qwen2.5-7B-Instruct
```

## Training pipeline (run in order)

```bash
# Activate the environment
source .venv/bin/activate

# Step 1 — build the RAG index (~1 min)
python -m sair.data.ingest_equations

# Step 2 — generate SFT + GRPO datasets (~10–30 min depending on problem count)
python -m sair.data.build_training_dataset

# Step 3 — supervised fine-tuning (~3–5 h on one A100)
python -m sair.train_sft_distiller

# Step 4 — GRPO refinement with judge-verified reward (~2–4 h)
python -m sair.train_grpo_distiller

# Step 5 — generate the final cheat sheet
python -m sair.scripts.generate_cheat_sheet --version v1

# Step 6 — evaluate against the held-out split
python -m sair.scripts.evaluate_cheat_sheet sair/cheat_sheets/v1.md
```

Use `tmux` to keep jobs running after SSH disconnects:

```bash
tmux new -s train
# run your commands
# Ctrl+B, D  →  detach
tmux attach -t train
```

## Key design decisions

- **Reward alignment**: the GRPO reward calls the *exact same* `judge_response` function from the cloned SAIR judge repo (loaded via `importlib.util`). The training signal cannot diverge from the grader.
- **Graceful degradation**: if the vector store is missing → retriever returns `[]`; if GRPO weights are missing → falls back to SFT; if both → uses the base model.
- **10 KB budget enforced at two levels**: per-cluster byte cap in `synthesize_cheat_sheet_entry` (1 200 B) and global `CheatSheet.render(max_bytes=10_000)` which drops low-priority sections.
- **LangGraph compliance**: `sair/graph.py` exposes both `run_pipeline_sequential` (used by the API, avoids the langgraph import) and `build_langgraph_app` (full `StateGraph` wiring for the multi-agent course rubric).
